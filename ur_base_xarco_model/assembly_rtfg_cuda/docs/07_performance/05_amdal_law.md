# Amdahl 定律与端到端分析

## Amdahl 定律

Amdahl 定律描述并行化的理论加速上限：

```
加速比 = 1 / ((1 - P) + P/N)

其中:
  P = 可并行化比例
  N = 并行度 (处理器数)
```

## CUDA IK 求解器的 Amdahl 分析

### 可并行化比例

| 阶段 | 总时间占比 | 可并行化? | 加速类型 |
|------|-----------|----------|---------|
| 273 独立 IK 求解 | ~98% | ✅ | 数据并行 (273×) |
| 数据打包 H2D | ~1% | ❌ (受限于传输) | 串行 |
| D2H 回读 | ~0.5% | ❌ (受限于传输) | 串行 |
| 碰撞检测 (FCL) | ~0.3% | ❌ (串行库) | 串行 |
| 创建 ROS 消息 | ~0.2% | ❌ (ROS 限制) | 串行 |

### 理论加速上限

```
P = 0.98 (98% 可并行化)
N = 273 (273 个目标并行)

Amdahl: 加速比 = 1 / ((1-0.98) + 0.98/273)
                  = 1 / (0.02 + 0.0036)
                  = 1 / 0.0236
                  = 42.4×
```

**42.4× 的理论上限** 远低于实测的 960×。这是因为 Amdahl 定律假设并行部分只能获得 N 倍加速，但我们的 GPU 加速不仅是并行：

### 实际加速来源分解

```
960× 加速比 = 273× (批处理并行) × 3.5× (其他优化)

其他优化包含:
  - 寄存器/共享内存优化: ~1.5×
  - 常量内存广播: ~1.2×  
  - 编译器优化 (-O3 -arch=sm_89): ~1.1×
  - 更少迭代 (6.7 vs 9.2): ~1.37×
  - 减轻 CPU 调度开销: ~1.3×
```

## 瓶颈分析

### 当前流水线瓶颈

```
GPU Kernel (6.434ms) ─┬── 93.5% ── ★ 主瓶颈
CPU 碰撞检测 (5ms)    ├── 串行，不可并行化
CPU 轨迹生成 (3ms)    └── 可优化
```

### 优化方向

1. **GPU-CPU 流水线重叠**: 将碰撞检测移到 GPU（已有 CUDA 版本计划）
2. **双流并发**: 使用 2 个 cudaStream 交替执行 IK 和碰撞检测
3. **CUDA Graph**: 捕获重复的求解流程，减少 Kernel Launch 开销
4. **核函数融合**: 将 continuity_cost 合并到主 kernel 末尾

## 总结

| 定律 | 预测上限 | 实测 | 差异原因 |
|------|---------|------|---------|
| Amdahl 定律 | 42.4× | 960× | 低估了架构差异 (GPU 微架构优化) |
| Gustafson 定律 | 273× | 960× | 问题规模不变，Gustafson 不适用 |
| 实测加速 | - | 960× | 批处理并行 + GPU 微架构优化 |
