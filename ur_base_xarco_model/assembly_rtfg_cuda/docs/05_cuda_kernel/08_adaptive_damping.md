# 自适应阻尼策略

## 概述

阻尼因子 $\lambda$ 是 DLS (Damped Least Squares) 方法的核心参数。过小的 $\lambda$ 导致步长过大、振荡不收敛；过大的 $\lambda$ 导致收敛缓慢。本包实现了根据当前误差动态调整 $\lambda$ 的自适应策略。

**源码位置**: `cuda_kernels.cu:176-192`

## 自适应公式

### 基准阻尼

```cpp
// cuda_kernels.cu:177-186
double pos_err = s_pos_err;
if (pos_err > 0.1) {
    // 远距离: 与误差成比例的中等阻尼
    s_lambda = fmax(2e-3, 8e-3 * (pos_err / 0.05));
    s_lambda = fmin(s_lambda, 0.15);
} else {
    // 近距离: 较高基础阻尼防止振荡
    s_lambda = 2e-3 + 8e-3 * (pos_err / 0.05);
}
```

### 数学表达

$$\lambda = \begin{cases}
\max(2\times 10^{-3}, 8\times 10^{-3} \cdot \frac{\|e\|}{0.05}), & \|e\| \leq 0.1 \\
\min(0.15, \max(2\times 10^{-3}, 8\times 10^{-3} \cdot \frac{\|e\|}{0.05})), & \|e\| > 0.1
\end{cases}$$

```mermaid
graph LR
    PE[position error] -->|>0.1| LM[λ = fmax(2e-3, 8e-3×err/0.05)<br/>cap at 0.15]
    PE -->|≤0.1| LN[λ = 2e-3 + 8e-3×err/0.05<br/>higher floor prevents oscillation]
    LM -->|stagnation>5| LS[λ × (1+0.3×(stag-5))<br/>cap at 0.5]
    LN -->|stagnation>5| LS
```

### 参数选择理由

| 参数 | 值 | 理由 |
|------|-----|------|
| 最小 λ | 2e-3 | 确保 Hessian 在近距离仍然正定 |
| 参考距离 | 0.05 m (5 cm) | 位置误差的典型收敛范围 |
| λ 缩放 | 8e-3/0.05 = 0.16/m | 使得 λ 在误差 5cm 时为 8e-3 |
| 远距离上限 | 0.15 | 在误差较大时不过度阻尼 |
| 停滞提升上限 | 0.5 | 最大阻尼以防止发散 |

## 停滞提升

当求解陷入停滞（连续 `>5` 次迭代无改善），$\lambda$ 被额外提升：

```cpp
// cuda_kernels.cu:188-191
if (s_stagnation > 5) {
    s_lambda *= (1.0 + 0.3 * (s_stagnation - 5));
    s_lambda = fmin(s_lambda, 0.5);
}
```

### 停滞提升曲线

| 停滞次数 | λ 乘数 | 最终 λ (假设基准 0.01) |
|---------|-------|---------------------|
| 0-5 | 1.0× | 0.01 |
| 6 | 1.3× | 0.013 |
| 10 | 2.5× | 0.025 |
| 15 | 4.0× | 0.04 |
| 20 | 5.5× | 0.055 |
| 25+ | (停滞恢复触发) | — |

## 与其他实现对比

### CPU 版本 (`cuda_ik_solver.cu`)

CPU 端 `initialize()` 上传的 `c_lambda_params` 包含基准 λ 配置：

```cpp
double h_lambda[4] = {
    cfg.lambda,     // [0] 基准 λ (来自配置)
    0.1,            // [1] 远距离上限
    5e-4,           // [2] 近距离下限 (GPU 实际使用 2e-3, 更保守)
    0.05            // [3] 缩放参考距离
};
```

### GPU 与 CPU 差异

本功能包当前在 Kernel 内部**硬编码**了 λ 参数，未使用 `c_lambda_params`：

| 参数 | GPU Kernel (硬编码) | CPU 配置传参 | 差异原因 |
|------|-------------------|-------------|---------|
| 最小 λ | 2e-3 | cfg.lambda (~1e-3) | GPU 更保守防止发散 |
| 近距离公式 | 2e-3 + 8e-3×e/0.05 | 5e-4 + 5e-3×e/0.05 | GPU 更高基础阻尼 |
| 远距离上限 | 0.15 | 0.1 | GPU 适度放宽 |
| 停滞上限 | 0.5 | 0.5 | 相同 |

**未来优化**：可以从 `c_lambda_params` 常量内存读取这些参数，使 GPU Kernel 参数可配置。

## 阻尼效果分析

### 典型收敛轨迹

```
误差 0.10 m ── λ = 0.016 (远距离, 中等阻尼)
   ↓
误差 0.05 m ── λ = 0.008 (参考距离)
   ↓
误差 0.02 m ── λ = 0.005 (近距离, 较高基础)
   ↓
误差 0.01 m ── λ = 0.0036 (收敛阶段)
   ↓
误差 0.005 m ── λ = 0.0028 (接近容差)
   ↓
误差 < 0.001 m ── 收敛
```

### 关键设计权衡

- **高阻尼 (λ↑)**: 步长小, 稳定但慢
- **低阻尼 (λ↓)**: 步长大, 快但可能振荡 
- **自适应**: 远距离快速下降 → 近距离精细调整
