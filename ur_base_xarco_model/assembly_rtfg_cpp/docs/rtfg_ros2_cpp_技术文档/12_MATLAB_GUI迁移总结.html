12. MATLAB GUI 迁移总结

从 MATLAB realtime_trajectory_fit_gui 到 ROS2 Web Dashboard (rtfg_gui) 的完整迁移回顾

1. 迁移背景与目标

原系统：MATLAB realtime_trajectory_fit_gui 是一个基于 uifigure 的桌面 GUI 应用程序，用于 UR10 机械臂的实时轨迹拟合与控制。它整合了环境位姿调整、3D 轨迹参数调整、统一 YAML 保存/重载以及 URDF 位姿回写功能。核心功能包括：

  • 3D 场景渲染与轨迹预览（基于 MATLAB uiaxes）
  • 六自由度环境盒子（block_with_basin）位姿调整
  • 铲斗轨迹参数调整（壁距、泥面高度、入泥角、深度等）
  • 运动学 IK 拟合与碰撞检测
  • YAML 配置文件读写

局限性：

  • 强依赖 MATLAB Runtime 环境，用户必须安装 MATLAB 或 MCR
  • 仅限桌面运行，无法远程访问或集成到自动化流水线
  • GUI 性能受限于 MATLAB 解释器与事件循环
  • 可视化能力局限于 MATLAB 图形系统，难以与现代 Web 3D 技术对接

迁移目标：

  • 构建一个跨平台、无 MATLAB 依赖的 Web Dashboard
  • 通过 RESTful API + Socket.IO 实时通信替代 MATLAB 回调架构
  • 使用 Three.js 3D 引擎替代 MATLAB uiaxes 渲染
  • 与 ROS2 C++ 求解器节点 (rtfg_solver_node) 深度集成
  • 保留全部原始功能，同时获得更好的可扩展性和可访问性

MATLAB GUI 使用 uifigure 创建主窗口，尺寸为 1500x880，采用 uigridlayout 划分为三个主要区域：

3.2 组件详解

3.2.1 Flask Web Server (app.py)

基于 Flask 的 Web 服务器，提供 7 个 RESTful API 端点和 Socket.IO 实时通信：

端点方法功能对应 MATLAB
/GET主页面 (index.html)buildMainUi
/trajectory_paramsGET轨迹参数页面openTrajectoryWindow
/environment_paramsGET环境位姿页面openEnvironmentWindow
/api/load_configPOST从 YAML 重载配置onReloadYaml
/api/fit_previewPOST执行轨迹拟合预览onTrackTcpTrajectory
/api/execute_cachedPOST执行缓存轨迹onPlayTcpTrajectory
/api/statusGET查询节点状态—
/api/save_configPOST保存 YAML 配置onSaveYaml
/api/save_urdfPOST保存 YAML + URDFonSaveYamlAndUrdf
/api/move_to_startPOST移动到起始位姿onMoveToTrajectoryStart

3.2.2 ROS2 Client Layer (ros_client.py)

在后台线程中运行 rclpy.spin，封装三个 ROS2 Service 客户端：

  • load_config(yaml_path) - 调用 /rtfg/load_config
  • fit_preview(params) - 调用 /rtfg/fit_preview，返回完整轨迹、碰撞数据和性能指标
  • execute_cached() - 调用 /rtfg/execute_cached

所有 Service 调用均带超时保护：加载配置 10s，拟合预览 120s，执行缓存 10s。

3.2.3 Three.js 3D Viewer (viewer.js)

使用 Three.js 的 URDFLoader 加载 UR10 模型，提供：

  • 暗色主题场景（GitHub Dark 背景 #0d1117）
  • 轨道控制（OrbitControls）支持旋转/缩放/平移
  • TCP 轨迹线 + 位置球体 + 方向箭头
  • 碰撞标记（红色半透明球体）
  • 四种预设视角：Isometric / Top / Front / Side
  • 世界坐标系、轨迹箭头、连杆坐标系、碰撞标记四个显隐切换

3.2.4 Bootstrap UI (index.html + app.js)

单页应用架构，主要包括：

  • 顶部工具栏：8 个按钮 + 状态指示灯
  • 视图控制栏：视角选择 + 4 个复选框
  • 3D 视口 + 右侧面板（状态、性能指标、响应窗口）
  • Bootstrap Modal 实现轨迹参数/环境位姿弹窗，滑块实时同步
  • Socket.IO 实时更新状态和性能指标
  • 键盘快捷键：Ctrl+Enter 触发轨迹拟合

7. 使用说明

7.1 一键启动

项目提供了 rtfg_one_click.sh 脚本，自动完成所有启动步骤：

cd /path/to/gui/scripts
chmod +x rtfg_one_click.sh
./rtfg_one_click.sh

脚本执行流程：

  1. 自动检测并加载 ROS2 环境 (Humble / Jazzy / Rolling)
  2. 加载工作空间 overlay (本项目 install/setup.bash)
  3. 通过 ros2 launch 启动 rtfg_gui.launch.py，同时启动 solver node 和 GUI 服务器
  4. 3 秒后自动在浏览器中打开 http://localhost:5000

7.2 手动启动

# Step 1: Source environments
source /opt/ros/humble/setup.bash
source /path/to/workspace/install/setup.bash

# Step 2: Launch solver + GUI
ros2 launch rtfg_gui rtfg_gui.launch.py port:=5000 debug:=false launch_solver:=true

# Step 3: Open browser at http://localhost:5000

7.3 环境变量

变量名默认值说明
RTFG_GUI_HOST0.0.0.0Flask 监听地址
RTFG_GUI_PORT5000Flask 监听端口
RTFG_GUI_DEBUGfalseFlask 调试模式

7.4 操作流程

  1. 加载配置：点击"从 YAML 重载"，从 YAML 文件加载上次保存的参数
  2. 调整轨迹参数：点击"轨迹调整"，在模态框中修改壁距、泥面高度、入泥角度等参数，点击 Apply
  3. 调整环境位姿：点击"盒子移动"，在模态框中修改 X/Y/Z 位置和 Roll/Pitch/Yaw 角度，点击 Apply
  4. 轨迹拟合：点击"轨迹拟合"按钮（或按 Ctrl+Enter），等待后端 IK 求解和碰撞检测完成
  5. 执行轨迹：拟合成功后"运行"按钮启用，点击后提交执行指令到 /rtfg/execute_cached
  6. 保存配置：点击"保存 YAML"将当前参数保存到文件，或"保存 YAML + URDF"同时回写 URDF
