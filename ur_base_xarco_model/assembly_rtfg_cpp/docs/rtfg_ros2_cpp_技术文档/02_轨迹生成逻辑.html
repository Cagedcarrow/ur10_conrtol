2. 轨迹生成算法

YOZ 三段轨迹构造、参数限幅、2D→3D 提升、环境位姿变换、TCP 姿态插值 — C++ 实现

1. 研究背景与目标

轨迹生成模块是 assembly_rtfg_cpp 的核心上游环节，它负责从高层任务参数（切入偏移、挖掘深度、切入角度等）生成一条完整的、可供下游 IK 求解器逐点求解的离散轨迹。该模块解决了以下关键问题：

  • 从抽象参数到几何路径：将料斗局部坐标系下的 YOZ 平面参数（left_wall_offset、mud_height、depth、theta_deg 等）映射为连续的离散点列。
  • 工程安全限幅：所有用户输入参数均需经过物理边界限幅，防止轨迹超出料斗内壁或机械臂关节空间。
  • 维度提升与坐标变换：从 YOZ 平面 2D 轨迹提升到料斗局部 3D 空间，再经环境位姿变换映射到机器人基坐标系。
  • TCP 姿态构造：位置轨迹点本身不包含姿态信息，需根据切线方向与连续性约束构造合理的 TCP 目标姿态（含 slerp 过渡），减少 wrist 关节突变。

第一版 ROS2 C++ 实现直接沿用了 MATLAB runtime 的 YAML 参数字段和相同的三段式轨迹几何结构，便于与旧工作流逐项对照验证。参数读取入口为 loadRuntimeConfig，同时支持 MATLAB runtime YAML 的嵌套字段和简化字段，允许旧 YAML 直接复制进新包。

2.3 料斗几何常量

第一版在 trajectory_generator.cpp 内维护一组与 MATLAB 版一致的料斗局部几何近似，用于轨迹参数限幅和 tool-basin 碰撞盒构造。这些值不是 URDF mesh 的几何解析结果，而是算法层的保守包络，便于控制狭小空间中的 clearance。

world_origin    = [-0.135, 0.0, 0.25]
outer_length_x  = 0.37
outer_width_y   = 0.50
outer_height_z  = 0.18
wall_thickness  = 0.003
inner_length_x  = outer_length_x - 2 * wall_thickness
inner_width_y   = outer_width_y  - 2 * wall_thickness
floor_z         = world_origin.z

2.4 YOZ 三段轨迹构造

MATLAB 版核心轨迹是料斗局部 YOZ 平面上的三段曲线。ROS2 第一版保持相同结构，依次生成斜线切入段、圆弧挖掘段和出泥提升段。

2.4.1 斜线切入段

切入点由横向偏移和泥面高度决定：

y_entry = inner_y_min + left_wall_offset
z_mud   = floor_z + mud_height
p_entry = [y_entry, z_mud]

切入方向由切入角控制：

theta     = deg2rad(theta_deg)
dir_entry = [cos(theta), sin(theta)]
p_approach = p_entry - approach_len * dir_entry

第一段从 p_approach 线性插值到 p_entry，默认 30 点。它对应机械臂从安全姿态逐步进入料斗目标平面，主要保证尖端进入泥面前的运动方向连续。

2.4.2 圆弧挖掘段

圆弧半径由深度和切入角反算：

radius = depth / (1 - cos(abs(theta)))

圆心位置：

a1    = theta - pi / 2
a2    = -pi / 2
center = p_entry - radius * [cos(a1), sin(a1)]

圆弧从角度 a1 插值到 a2，默认 140 点。该设计的效果是：初始切向和斜线切入方向一致，圆弧末端转为接近竖向出泥方向，同时 depth 控制圆弧最低点相对泥面的垂直侵入量。

2.4.3 出泥提升段

圆弧终点 p_arc_end 之后，轨迹保持同一个 y 坐标，沿 z 方向竖直抬升：

lift_len = min(z_motion_max - p_arc_end.z, max(0.06, 1.20 * radius))

默认 90 点。提升长度既受料斗上沿约束，也和圆弧半径相关，避免圆弧很大时提升不足。

2.5 2D 到 3D 提升

YOZ 点 [y, z] 被提升为料斗局部 3D 点：

P_local = [x_plane, y, z]

x_plane 是固定作业平面，用于控制铲尖在料斗长向上的位置。这样保留了 MATLAB 版"先在 YOZ 平面设计轨迹，再投射到 3D 工作平面"的思路，算法调试时只需要看二维几何就能理解深度、角度和切入偏移。

2.6 环境位姿变换

料斗局部轨迹通过 base_jizuo_to_block_with_basin_frame 位姿变换到机器人基坐标：

T_base_basin = transl(x, y, z) * Rz(yaw) * Ry(pitch) * Rx(roll)
P_base       = T_base_basin * P_local

这个变换保持了 MATLAB environmental_model 里的可编辑入口：URDF 中的 base_jizuo_to_block_with_basin_frame 是长期场景装配基准，ROS2 版本不重新发明场景坐标，而是继续沿用这个基准。

2.7 TCP 姿态构造

轨迹点只定义了尖端位置，还需要构造 TCP 目标姿态。第一版采用局部切线策略，分两个阶段处理：

斜线和圆弧阶段

  • 计算相邻点切向向量 tangent。
  • TCP X 轴取 -tangent，让工具尖端方向反向对齐运动方向。
  • TCP Z 轴尽量靠近世界上方，先去除 X 轴方向分量，再归一化。
  • TCP Y 轴由 Z × X 叉积得到，保证右手系。

出泥提升阶段

  • TCP Z 轴固定朝上。
  • X 轴根据圆弧末段的水平 heading 保持连续。
  • Y 轴由 Z/X 叉积构造。

姿态过渡窗口

圆弧到出泥之间增加姿态过渡窗口，避免 wrist 关节在圆弧末端突然承担大角速度：

alpha = quinticBlend(i / (transition_steps - 1))
R     = slerp(R_arc_end, R_lift, alpha)

这种做法比直接切换姿态更适合狭小空间，因为 wrist 关节不会在圆弧末端突然承担大角速度。

2.8 段名标签

每个目标点都带有段名，用于碰撞审计时定位问题发生的阶段：

  • 斜线 — 切入段
  • 圆弧 — 挖掘段
  • 出泥 — 提升段

段名会进入 TrajectoryResult.playback_segment_names，并在碰撞审计发现问题时回传，便于知道碰撞发生在切入、挖掘还是提升阶段。

4. 结果分析

4.1 参数限幅的有效性

所有 7 项用户参数均通过物理边界限幅，保证了轨迹始终位于料斗内壁的安全包络内。approach_len 上限 0.30 m 防止切入段过长导致机械臂伸展至奇异位形；theta_deg 限幅到 [-60, -5] 度确保圆弧切入角既有足够的水平分量又不会过于平缓；depth 限幅防止挖掘过深引发碰撞或关节超限。

4.2 三段轨迹的几何连续性

斜线切入段与圆弧段在 p_entry 处切向连续（圆弧初始角 a1 = theta - pi/2 与斜线方向 dir_entry 对齐），圆弧末端与出泥提升段在 y 坐标上保持一致，整体位置轨迹为 G1 连续。姿态方面通过 slerp 过渡窗口进一步保证了 wrist 关节角速度的平滑性。

4.3 与 MATLAB 版本的差异分析

第一版有两处工程化差异：

  • 交互方式：MATLAB GUI 可以在界面上交互调整参数；ROS2 第一版通过服务请求传参，不提供 GUI 控件。
  • 姿态过渡：ROS2 版为圆弧到出泥增加了显式姿态 slerp 过渡，目的是降低狭小姿态下 wrist 突变风险。这不改变位置轨迹，但会影响 IK 姿态目标，带来更平滑的关节运动。

4.4 几何回归验证结论

默认 YAML 配置下，ROS2 C++ 版生成 273 个目标点，approach_start、entry、arc_end 等关键几何特征点坐标与 MATLAB 版一致（四舍五入到小数点后五位），圆弧半径 0.389 m、垂直侵入量 0.052 m 均在预期范围内，验证了轨迹生成逻辑的移植正确性。

5. 结论与改进效果

5.1 当前效果总结

  • 成功将 MATLAB 版 YOZ 三段轨迹生成逻辑移植为 ROS2 C++ 实现，273 个离散目标点的几何回归验证通过。
  • 参数限幅机制完善，所有 7 项用户参数均有物理边界保护，防止轨迹超出料斗安全包络。
  • 2D→3D 提升 + 环境位姿变换 + TCP 姿态构造流水线完整，支持从高层任务参数到基坐标系下完整位姿轨迹的自动生成。
