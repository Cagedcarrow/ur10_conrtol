3. 实时轨迹拟合与运动学求解架构

DLS IK 算法、权重调度、连续预测求解器、多候选搜索、等效关节角对齐、五次样条回放

3.1 研究背景与目标

本模块解决的核心问题是：给定一串目标 TCP 齐次变换序列，求解出连续的六轴关节角轨迹，使得机器人末端能够平滑、无碰撞地跟随指定路径。

输入为齐次变换序列：

T_target[i] = [R_target[i], p_target[i]]

输出为连续关节角序列：

q[i] = [q1, q2, q3, q4, q5, q6]

约束条件包括：

  • TCP 位置误差小于 ik_position_tolerance
  • 姿态误差在当前权重阶段允许范围内
  • 相邻点关节变化连续，避免跳变
  • 工具和机器人本体、料斗壁之间满足 clearance 门限
  • 最终能生成可回放的 trajectory_msgs/JointTrajectory

这个问题不是简单的单点 IK 求解，而是狭小空间中的连续轨迹拟合。单点 IK 可行不代表整条轨迹可行，必须同时考虑等效关节角对齐、速度连续性和局部碰撞。这一架构的目标是在保证实时性的前提下，为挖泥类狭小空间作业提供鲁棒的轨迹求解能力。

3.2 算法与架构设计

3.2.1 运动学模型

robot_model.cpp 使用 urdfdom 解析 assembly_rtfg_solver.urdf，从 sensor_shovel_tcp 反向追踪到 base_jizuo，再正向建立运动学链：

base_jizuo
  -> ...
  -> ur10_shoulder_pan
  -> ur10_shoulder_lift
  -> ur10_elbow
  -> ur10_wrist_1
  -> ur10_wrist_2
  -> ur10_wrist_3
  -> sensor_shovel
  -> sensor_shovel_tcp

固定关节保存为段的 origin，可动 revolute/continuous 关节保存轴向、上下限和 q_index。正向运动学时按段累乘：

T = T * joint_origin
if joint movable:
  T = T * Rot(axis, q)
TCP 位置说明：末端 TCP 是 sensor_shovel_tcp，不是 ur10_wrist_3。这一点很重要，因为 MATLAB 的目标就是铲尖点拟合；如果把 wrist link 当 tip，所有几何和碰撞解释都会偏移一个工具长度。

3.2.2 姿态误差定义

位置误差采用简单的向量差：

e_p = p_target - p_current

姿态误差使用旋转对数（rotation logarithm）将旋转矩阵映射到三维向量：

R_err = R_current^T * R_target
w_local = log(R_err)
e_r = R_current * w_local

将局部旋转误差转换到世界方向后，位置和姿态组成 6 维误差向量：

e = [e_p; e_r]

3.2.3 阻尼最小二乘 (DLS) IK 求解器

核心迭代求解公式：

H = J^T W^T W J + lambda I
g = J^T W^T W e
dq = H^-1 g
q_next = clamp(q + dq)

其中：

  • J 是 6x6 TCP 雅可比矩阵
  • W 是任务空间权重矩阵
  • lambda 是自适应阻尼系数
  • clamp 保证关节角不越过 URDF 限位

自适应阻尼策略（继承自 MATLAB MEX 基线）：

  • 离目标较远时增大阻尼，避免关节跳变
  • 接近目标时降低阻尼，提高收敛精度
  • 单次 dq 范数限制在 0.45 rad 内，避免狭小空间里一步跨过碰撞边界

solveSinglePose 函数的关键参数：

参数值说明
阻尼 lambda[5e-4, 0.1]自适应范围
最大迭代次数60单次求解迭代上限
停滞检测步数8连续无改进则提前终止

3.2.4 几何雅可比实现

当前代码中 numericJacobian 接口已按几何雅可比计算，不再需要每列做一次 FK 差分。对第 i 个转动关节：

Jv_i = axis_i x (p_tcp - p_joint_i)
Jw_i = axis_i

这样每次 IK 迭代只需要沿运动学链前向累乘一次即可得到所有关节轴和 TCP 点。接口名称保留为 numericJacobian 是为了减少迁移范围，后续可重命名为 geometricJacobian。

3.2.5 多阶段权重调度

狭小姿态下，严格要求位置和姿态同时完全贴合，容易造成第一锚点或圆弧末端失败。第一版使用四阶段权重策略：

阶段位置权重姿态权重姿态误差上限
11.00.2030 deg
21.00.1045 deg
31.00.0370 deg
41.00.00不限制

策略含义：优先尝试严格姿态求解；如果不可行，逐步放松姿态约束，确保尖端位置先跟上轨迹。对挖泥类任务，尖端位置精度和碰撞安全通常比工具姿态的微小偏差更关键。

3.2.6 多种子搜索策略

每个目标点按以下顺序尝试种子：

  1. q_prev：上一点结果，最符合连续轨迹假设
  2. home_q：URDF 中的初始姿态
  3. zero：零位参考
  4. wrist 5/6 的 ±2π 等效展开
  5. 固定随机全局种子（中位 + 96 均匀随机，rng=42）

种子调度函数：

函数算法关键参数
buildSeedList3 基种子 × 9 包裹组合 (J5, J6 ±2π)去重容差 1e-9 rad
buildGlobalSeedList中位 + 96 均匀随机 (rng=42)关节限位内均匀分布

这种顺序来自 MATLAB MEX 的经验：绝大多数连续点应由 q_prev 成功，只有在接近奇异、姿态翻转或碰撞门限附近时才需要展开等效解。

3.2.7 等效关节角对齐

UR10 的 wrist 关节存在 q 和 q ± 2π 等效问题。单点 IK 可能返回几何上正确但离上一点很远的解。第一版通过 alignToReference 把候选角对齐到参考值：

q_ref = q_prev + dq_prev

对每个关节尝试 q + 2πk，选择离 q_ref 最近且满足 joint limit 的值。这样可以避免轨迹中出现视觉上突然绕一整圈的关节跳变。

3.2.8 连续性代价排序

候选解排序使用的代价函数：

cost = norm(wrap(q_candidate - q_prev))
       + 0.65 * norm(wrap(q_candidate - q_prev) - dq_prev)

第一项控制位置连续性（避免大关节步长），第二项控制速度趋势连续性（避免相邻段之间来回摆动）。加权系数 0.65 由 MATLAB MEX 经验调优得到。

3.2.9 延迟碰撞验证

碰撞检测比一次误差计算昂贵得多。第一版不会对每个未收敛候选都做 FCL 距离查询，而是先用 IK 容差门控：

if pos_err <= tolerance and rot_err <= orient_limit:
  evaluateConfiguration(...)

只有 IK 误差达标的候选才进入碰撞检查。这样既保留了碰撞安全保障，又减少了大量无意义的 FCL 调用。

3.2.10 Relaxed-orientation Fallback

如果严格姿态失败，但存在一个位置误差达标且 clearance 合格的候选，第一版允许接受 relaxed-orientation fallback。它的作用是解决 MATLAB MEX 中常见的首锚点或复杂狭小姿态失败问题：

  • 位置必须达标
  • 碰撞 clearance 必须达标
  • 姿态误差可以超过严格阶段上限
  • 返回结果中会记录 fallback 原因，便于后续审计
设计取舍：这不是忽略姿态，而是在工具尖端位置和安全性优先的任务里，给狭小空间留出可行解的空间。

3.2.11 五次样条回放轨迹生成

锚点求解后，不能直接把 anchor 点作为执行轨迹。第一版按三类步长估计插值数量：

joint_step <= 0.70 deg
pos_step   <= 0.003 m
rot_step   <= 0.30 deg

每两个锚点之间插入 4 到 32 个 playback 点，插值参数使用五次多项式（quintic blend）：

s(t) = 10t^3 - 15t^4 + 6t^5
q(t) = q_a + s(t) * wrap(q_b - q_a)

五次样条的端点一阶和二阶导数为 0，相比线性插值更适合机器人回放场景，能够保证启停平滑。

3.4 结果分析

基于当前拟合行为的验证结果，可以得出以下分析：

  • 算法链路完整性验证通过：从 URDF 解析、运动学建模、多阶段权重调度、多种子 IK 求解、等效关节角对齐、连续性代价排序、延迟碰撞验证到五次样条回放，整条管线可以完整运行。
  • IK 求解能力达标：273 个锚点全部完成 IK 求解，表明 DLS 自适应阻尼算法配合多种子搜索策略能够覆盖全部轨迹点，未出现无法收敛的断点。
  • 碰撞安全仍需改进：尽管 IK 求解全部成功，碰撞审计发现 tool-basin penetration，说明默认轨迹参数或工具几何定义仍存在优化空间。需要在后续迭代中增加避障策略或调整轨迹参数。
  • 瓶颈定位：当前主要瓶颈不在于 IK 求解本身，而在于碰撞检测后的参数调优环节。默认 YAML 配置下的轨迹虽然运动学可行，但未满足安全 clearance 门限。

3.5 结论与改进效果

本模块成功将 MATLAB MEX 的轨迹拟合与运动学求解能力迁移至 C++ ROS2 架构，实现了以下改进：

  • 架构可扩展性：从 MATLAB MEX 共享库迁移为独立 ROS2 C++ 节点，支持与 ROS2 生态原生集成（参数服务器、主题通信、生命周期管理）。
  • 实时性基础：几何雅可比替代数值差分雅可比，减少每次 IK 迭代的计算量；延迟碰撞验证机制减少了无效 FCL 调用；五次样条回放保证轨迹平滑性。
  • 代码可维护性：模块化设计将 IK 求解器、碰撞检测器、机器人模型、数学工具分离为独立编译单元，便于单模块测试和替换。
  • 算法链路完整性：完整保留了 MATLAB MEX 中经过验证的 DLS 自适应阻尼、四阶段权重调度、多种子搜索、等效关节角对齐、连续性代价排序等核心策略。

后续优化方向：

  • 增加避障策略或调整默认轨迹参数以解决 tool-basin penetration 问题
  • 完善性能计时基础设施，建立与 MATLAB MEX 基线的定量加速比对比
  • 将 numericJacobian 重命名为 geometricJacobian，消除命名歧义
  • 进一步优化碰撞检测的 BVH 树缓存策略
