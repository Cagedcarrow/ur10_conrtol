9. 核心算法公式与参数说明

DLS 求解公式、权重调度策略、连续性代价函数、调参顺序与典型现象诊断手册

9.1 研究背景与调参目标

assembly_rtfg_cpp 的调参不是只让 IK 返回成功，而是同时满足四个条件：

  1. TCP 位置轨迹符合挖掘几何 — 轨迹必须贴合料斗形状，避免工具与料斗壁碰撞。
  2. 工具姿态变化连续 — 不在 wrist 处产生突变，保证关节轨迹平滑可回放。
  3. 工具、机器人本体和料斗之间保持 clearance — 碰撞安全距离始终高于阈值。
  4. 关节轨迹能被 ros2_control 回放 — 步长、速度、加速度不超出硬件限制。

因此，调参时不能只看 success，还要看全套诊断指标：

指标含义理想范围
anchor_count锚点 IK 成功数接近总目标点数
playback_count回放插值成功数接近锚点数
max_anchor_qstep_deg锚点间最大关节步长< 5-10 deg
max_playback_qstep_deg回放最大关节步长< 2-5 deg
max_target_rot_deg目标姿态最大旋转角视轨迹而定
max_actual_rot_deg实际姿态最大旋转角视轨迹而定
min_self_clearance最小自碰撞距离≥ clearance_threshold
min_tool_body_clearance最小工具-本体距离≥ clearance_threshold
min_tool_basin_clearance最小工具-料斗距离≥ clearance_threshold
first_collision_object首个碰撞对象名称—
first_collision_segment首次碰撞所在段—
调参核心原则：避免同时大幅修改多个变量。每次只改一个参数，记录 success、首个碰撞对象、最小 tool-basin clearance 和总耗时，逐步逼近最优解。

9.2 核心算法与数学公式

9.2.1 轨迹几何公式

切入点计算

切入点确定了工具进入料斗的水平位置和垂直深度：

y_entry = inner_y_min + left_wall_offset
z_mud  = floor_z + mud_height
p_entry = [y_entry, z_mud]

left_wall_offset 越小，轨迹越靠近左壁；越大，轨迹越靠近右壁。若碰撞对象是 block_basin_left_wall，通常应增大该值；若碰撞对象是 block_basin_right_wall，通常应减小该值。

切入起点（Approach）

在切入点之前沿切入方向延伸一段距离，形成过渡段：

dir = [cos(theta), sin(theta)]
p_approach = p_entry - approach_len * dir

approach_len 影响进入料斗前的过渡长度。太短会让第一段姿态变化集中，太长可能让工具在进入前扫过料斗边缘。

圆弧半径

挖掘轨迹的圆弧半径由切入深度和切入角共同决定：

radius = depth / (1 - cos(abs(theta)))

这个公式意味着：

  • depth 越大，圆弧越深，穿入风险越高。
  • theta_deg 绝对值越小，分母越小，半径会急剧变大。
  • theta_deg 太接近 0 时，圆弧不再适合表达挖掘动作，因此第一版把它限制到 -60 到 -5 度。

垂直侵入量

vertical_penetration = z_mud - p_arc_end.z

这个值越大，挖掘动作越深，但 tool-basin 或 tool-body 碰撞风险也会增加。默认 YAML 的派生值约为 0.05212 m。

9.2.2 姿态公式

Track 姿态（斜线和圆弧阶段）

在斜线切入和圆弧挖掘阶段，工具姿态与轨迹切向对齐：

x_axis = -normalize(tangent)
z_axis = normalize(world_up - dot(world_up, x_axis) * x_axis)
y_axis = normalize(cross(z_axis, x_axis))
z_axis = normalize(cross(x_axis, y_axis))

这个姿态让工具尖端方向与轨迹切向相关，同时尽量使工具上方向接近世界上方。

Lift 姿态（出泥阶段）

出泥阶段工具姿态切换到固定上方向，避免 wrist 过度翻转：

z_axis = [0, 0, 1]
x_axis = -normalize(horizontal_heading)
y_axis = normalize(cross(z_axis, x_axis))
x_axis = normalize(cross(y_axis, z_axis))

这让工具出泥时姿态更稳定，避免继续沿圆弧切线导致 wrist 过度翻转。

姿态过渡（Slerp + Quintic Blend）

R = slerp(R_track_end, R_lift, quinticBlend(alpha))

quinticBlend 使用五次多项式混合，确保过渡开始和结束处的一阶、二阶导数都连续平滑。

9.2.3 DLS（Damped Least Squares）IK 求解公式

IK 求解器使用阻尼最小二乘法（DLS），通过数值雅可比迭代逼近目标位姿：

J = numericJacobian(q)         // 6×6 数值雅可比
e = poseError(T_desired, T_current)  // 6D 姿态误差
dq = J^T * (J * J^T + lambda^2 * I)^{-1} * e
q_new = q + dq

其中 lambda 是自适应阻尼因子（范围 [5e-4, 0.1]），在接近奇异位姿时自动增大以维持数值稳定性。迭代上限为 60 步，停滞检测窗口为 8 步。

9.2.4 连续性代价函数

多候选 IK 结果遴选时，综合考虑位置误差、姿态误差和与前一点的关节连续性：

cost = w_pos * pos_error + w_rot * rot_error + w_cont * continuity_cost
continuity_cost = sum(w_i * |q_i - q_prev_i|)

continuity_cost 使用加权绝对值距离，权重根据关节类型分配：大臂关节（J1-J3）权重较低，wrist 关节（J4-J6）权重较高，以鼓励 wrist 优先对齐上一位姿。

9.2.5 权重调度策略

四阶段权重本质是"姿态从严格到放松"：

// 阶段 1 — 严格姿态
[1.00, 1.00, 1.00, 0.20, 0.20, 0.20]
// 阶段 2 — 放松方向
[1.00, 1.00, 1.00, 0.10, 0.10, 0.10]
// 阶段 3 — 大幅放松方向
[1.00, 1.00, 1.00, 0.03, 0.03, 0.03]
// 阶段 4 — 仅位置约束
[1.00, 1.00, 1.00, 0.00, 0.00, 0.00]

前三个分量对应位置权重（固定为 1），后三个分量对应姿态方向权重（逐阶段递减）。当某些点姿态不可达时，仍尽量让尖端位置跟上轨迹。对挖掘类任务，尖端位置、连续性和碰撞安全通常优先于末端姿态完全等同。

9.4 调参顺序与诊断流程

9.4.1 先解决碰撞

如果 min_tool_basin_clearance < 0（即发生穿透）：

  1. 查看 first_collision_object 确定碰撞对象。
  2. 如果是左壁（block_basin_left_wall），增大 left_wall_offset。
  3. 如果是右壁（block_basin_right_wall），减小 left_wall_offset。
  4. 如果是底部，减小 depth 或增大 mud_height。
  5. 如果碰撞出现在 出泥 段，检查 lift 姿态和 x_plane 偏移。

9.4.2 再解决 IK 失败

如果日志显示某个目标位姿求解失败：

  1. 降低 depth，减小圆弧深度。
  2. 减小 approach_len，缩短过渡段。
  3. 检查 theta_deg 是否接近限幅边界（-60 到 -5 度）。
  4. 使用当前真实 joint state 作为 current_q，不要只用 URDF 初始值。
  5. 若失败集中在第一个点，说明起始姿态和目标段不连续，应先调整 approach。

9.4.3 最后优化速度

只有当轨迹无碰撞且 IK 成功后，性能数字才有比较意义。带碰撞的失败路径会触发更多诊断和回传，不适合当最终性能目标。

9.6 结果分析与典型现象

9.6.1 trajectory solved but collision audit found violations

含义：

  • IK 成功。
  • playback 轨迹生成成功。
  • 稀疏碰撞审计发现 clearance 违规。
  • 结果可用于诊断，但不应直接执行。

处理：按照 9.4.1 碰撞解决流程调整轨迹参数。

9.6.2 clearance below threshold

含义：

  • IK 误差达标。
  • FCL 检测到候选配置离障碍太近（小于 clearance_threshold）。
  • 该候选被过滤，求解器继续搜索其他种子。

影响：可能增加 IK 迭代次数，但不一定导致最终失败。

9.6.3 accepted relaxed-orientation fallback

含义：

  • 严格姿态权重阶段没有找到更好候选。
  • 位置误差达标。
  • 碰撞 clearance 达标。
  • 姿态误差被允许放松到下一阶段权重。

影响：这是权重调度机制的正常行为。如果频繁出现在同一区域，说明该区域姿态约束过紧。

9.6.4 KDL root inertia warning

含义：

  • KDL 不支持 root link inertia。
  • 当前链仍可构建。
  • 如需消除，增加 dummy root link。

影响：不影响求解正确性，可忽略。

9.7 结论与改进效果

9.7.1 当前效果总结

第 1 版 assembly_rtfg_cpp 的算法与参数体系已建立以下能力：

  • 轨迹几何生成：支持切入点、approach、圆弧、lift 完整挖掘轨迹参数的解析计算。
  • 姿态调度：Track/Lift 双模式姿态 + Slerp/Quintic 平滑过渡。
  • IK 求解：DLS 数值求解器 + 四阶段权重调度 + 连续性代价遴选。
  • 碰撞检测：自碰撞、tool-body、tool-basin 三层 FCL 检测管线。
  • 调参诊断：全套指标输出 + 典型现象解释 + 分步调参策略。

9.7.3 可执行条件检查清单

只有满足以下所有条件才应调用 /rtfg/execute_cached：

  • /rtfg/fit_preview 返回 success=true。
  • min_tool_basin_clearance >= clearance_threshold。
  • min_tool_body_clearance >= clearance_threshold。
  • min_self_clearance >= clearance_threshold。
  • playback 点数合理。
  • 最大关节步长没有异常突变。
安全警告：如果服务返回 success=false 但仍有 trajectory，这个 trajectory 只能用于诊断和 RViz 观察，不应执行。

9.7.4 推荐调参扫描步长

基于当前默认参数的 tool-basin penetration 问题，建议以下小步扫描策略：

left_wall_offset: 当前值附近 ±0.02 m
x_plane:          当前值附近 ±0.02 m
depth:            从当前值逐步减小 0.005 m
theta_deg:        每次调整 2 到 5 deg

每次只改一个变量，记录：

  • 是否 success。
  • 首个碰撞对象。
  • 最小 tool-basin clearance。
  • 总耗时。
