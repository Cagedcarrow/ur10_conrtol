6. 服务接口与运行验证

3 个 Service 定义、Launch 参数说明、Benchmark 使用方法、验证步骤、Topic 列表

6.1 研究背景与目标

将 MATLAB MEX 版本的轨迹生成与 IK 求解管线迁移到独立 ROS2 C++ 包 assembly_rtfg_cpp 后，需要建立一套完整的服务接口层，使外部节点（GUI、脚本、自动化测试）能够以标准 ROS2 Service/Action 机制调用求解管线。

本模块解决以下核心问题：

  • 配置加载标准化：将 runtime YAML 的读取、解析、校验封装为统一的服务接口，消除直接文件 I/O 的耦合
  • 轨迹求解服务化：将 IK 求解、碰撞检测、轨迹回放封装为可远程调用的 Service，支持异步 benchmark 和自动化测试
  • 执行通道建立：通过 FollowJointTrajectory Action 将求解结果下发到 ros2_control 控制器，打通 "求解 -> 执行" 闭环
  • 可视化与调试：发布 TCP 路径、碰撞 Marker、MoveIt DisplayTrajectory，支持 RViz 实时预览

属性值

6.2.4 Launch 配置

完整启动（含 RViz）：

ros2 launch assembly_rtfg_cpp rtfg_sim.launch.py launch_rviz:=true

无 RViz、无 MoveIt 的 smoke 测试模式：

ros2 launch assembly_rtfg_cpp rtfg_sim.launch.py launch_rviz:=false launch_moveit:=false

Launch 文件执行流程：

  1. 展开 assembly_rtfg.urdf.xacro 生成 URDF
  2. 启动 robot_state_publisher
  3. 在 /rtfg namespace 下启动 ros2_control_node
  4. 加载 joint_state_broadcaster
  5. 加载 joint_trajectory_controller
  6. 可选启动 move_group
  7. 可选启动 RViz
  8. 启动 rtfg_solver_node
设计决策：控制器使用 /rtfg namespace 是为了避免和当前工作区已有的全局 /controller_manager 冲突，确保多包共存场景下的命名隔离。

6.5 结果分析与瓶颈定位

6.5.1 已知运行警告

KDL 在启动时输出以下信息：

The root link base_jizuo has an inertia specified in the URDF,
but KDL does not support a root link with an inertia.

这是 KDL 对 root link 含有 inertia 的常见警告，不影响当前 FK/IK 链的建立。后续若需消除，可在 URDF 顶部增加 dummy root link，将 inertia 移至非 root link。

6.5.2 瓶颈分析

  • 碰撞检测性能：第一版 FCL 碰撞检测尚未应用 thread_local 缓存优化，每次碰撞查询均需重建 BVH，导致耗时高于 MEX 版本的缓存机制
  • IK 求解效率：阻尼最小二乘法 (DLS) 的 60 次迭代上限和 8 步停滞检测逻辑与 MEX 版本一致，但 ROS2 服务调用的序列化/反序列化开销增加了每次请求的 latency
  • tool-basin penetration：默认 runtime 参数下存在 tool-basin 穿透，表明参数空间仍需调优——这不是算法问题，而是参数配置问题
  • 稀疏审计策略：当前仅在锚点处进行碰撞审计，未对所有 playback 点做全量检测，存在潜在的漏报风险

6.6 结论与改进路线

6.6.1 当前效果总结

assembly_rtfg_cpp 包已成功建立完整的服务接口层和运行验证体系：

  • 3 个 Service 覆盖配置加载、轨迹拟合预览、缓存执行全流程
  • 5 个 Published Topic 提供可视化与性能监控能力
  • Launch 文件支持 RViz/MoveIt 可选启动，namespace 隔离避免与现有工作区冲突
  • 几何回归测试通过（target_poses=273、radius=0.389029、vertical_penetration=0.05212）
  • Safe 参数配置下可实现无碰撞轨迹执行

6.6.2 下一步验证与优化建议

  1. 参数调优：先调整默认 runtime 参数，使 /rtfg/fit_preview 返回无 tool-basin penetration 的结果
  2. 分工况 Benchmark：对无碰撞参数记录新的 benchmark 数据，与当前碰撞参数分开比较，建立性能基线
  3. 全量碰撞审计：增加对所有 playback 点的碰撞检测测试，确认稀疏审计策略没有漏报
  4. 自动化 Smoke Test：增加 ROS2 Launch 级别的自动化测试，验证 controller action 可正常执行
  5. 性能优化：进入 IK 后端和响应序列化的性能优化阶段，目标是在第二阶段超过历史 MEX 版本的性能
